About keytool

The keytool module allows you to

View a keystore file
Create a new keystore file
Open a keystore file
Save the keystore file
Change keystore password
Create a new keystore entry
Delete a keystore entry
Create a Certificate Request File

View a keystore file

Display the content of a keystore in a window.

You can open the keystore window by selecting Windows|View Keystore, or by clicking the icon in the toolbar.
The keystore window displays the keystore entries of the currently loaded keystore file.

The keystore entries are displayed in a table at the bottom of the window. The details of a keystore entry are shown at the upper part of the window. You can choose to view the keystore entry either in the Verbose, or RFC view mode.

Create a new keystore file

A new keystore file is opened in the keystore window.

A wizard guides you through creating a new keystore file. The wizard has following steps:

  1. Select a new file
  2. Specify a keystore password, and retype the keystore password

Finishing the wizard creates an empty keystore file in the filesystem, and opens the empty keystore file in the keystore window.

Open a keystore file

A keystore file is opened, its content is displayed in the keystore window.

You can load a key store file by selecting the open keystore icon in the toolbar.
Alternatively, you can select a file having the extension keystore, or jks in the explorer window, opening the context menu and selecting Tools|Load Key Store.

Moreover you can drag a keystore file to the keystore window, and keystore filename text field. You either select the dragging keystore file from within Netbeans explorer view, or from the OS explorer application.

A wizard guides you through opening a keystore file. The wizard has following steps.

  1. Select a keystore file
  2. Enter the keystore password

By finishing the wizard the keystore file is loaded, and its content is shown in the keystore window.

Save the keystore file

The keystore file viewed in the keystore window is saved.

You must store a keystore explicitly.

  • If you create a new keystore, it is not saved automatically.
  • If you create a new keystore entry or if you delete a keystore entry, you have to save the keystore explicitly.

Change keystore password

Changes the keystore password of the keystore which is opened.

A wizard guides you through changing the keystore password. The wizard has following steps:

  1. Confirm the keystore file name which is used for storing the changed keystore password.
  2. Enter the new keystore password, and confirm the new keystore password.

Finishing the wizard the keystore with the new password is saved on disk.

The next time opening the keystore file, you have to provide the wizard-entered keystore password.

Create a new keystore entry

A new keystore entry is creating in the current keystore window.

A wizard guides you through creating a new key entry. The wizard has following steps:

  1. Enter certificate basic information, like alias, validity of the key, etc.
  2. Enter identity informations, like firstname, lastname, etc.
  3. Confirm informations entered

By finishing the wizard a new keystore entry is created in the currently opened keystore file. The created keystore entry is self-signed automatically.

You must save the keystore file for making the change persistent.

Delete a keystore entry

The current selected keystore entry is deleted from the keystore entry file.

You must save the keystore file for making the change persistent.

Create a certificate request file

Create a certificate request file.

A wizard guides you through creating the certificate request file. The wizard has following steps:

  1. Select the certificate request file. You must select a new file, files already existing are not acceptable.
    1. Select the alias for which the certificate request is created.
    2. Select the signature algorithm for creating the certificate request.
    3. Enter the key password, alternatively you may reuse the store password.

A dialog window informs you whether the certificate request file has been created successfully.

Usually, you sent the created certificate request file to a certification authority for signing it.

Note: Currently there is no feature in the keytool for importing the signed certificate request. You still have to use the traditional keytool command.
