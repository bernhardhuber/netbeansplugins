Rule LooseCoupling::

TOC

Description

Avoid using implementation types (i.e., HashSet); use the interface (i.e, Set) instead
Message
Avoid using implementation types like ''{0}''; use the interface instead
Class
net.sourceforge.pmd.typeresolution.rules.LooseCoupling
Reference
-
ExternalInfoUrl
http://pmd.sourceforge.net/rules/typeresolution.html#LooseCoupling
TypeResolution
true
Priority
3

Example

Example code snippet of this rule. 

import java.util.ArrayList;
import java.util.HashSet;
public class Bar {
 // Use List instead
 private ArrayList list = new ArrayList();
 // Use Set instead
 public HashSet getFoo() {
  return new HashSet();
 }
}

Rule CloneMethodMustImplementCloneable::

TOC

Description

The method clone() should only be implemented if the class implements the 
Cloneable interface with the exception of a final method that only throws 
CloneNotSupportedException. This version uses PMD's type resolution facilities, 
and can detect if the class implements or extends a Cloneable class
Message
clone() method should be implemented only if implementing Cloneable interface
Class

 public Object clone() throws CloneNotSupportedException {
  return foo;
 }
