Rule NPathComplexity::

TOC

Description

   The NPath complexity of a method is the number of acyclic execution paths through that method.
   A threshold of 200 is generally considered the point where measures should be taken to reduce complexity.
Message
The method {0}() has an NPath complexity of {1}
Class
net.sourceforge.pmd.rules.design.NpathComplexity
Reference
-
ExternalInfoUrl
http://pmd.sourceforge.net/rules/codesize.html#NPathComplexityh
TypeResolution
-
Priority
-

Example

Example code snippet of this rule. 
 
 public class Foo {
  void bar() {
   // lots of complicated code
  }
 }

Description
The npath reporting threshold

Rule CyclomaticComplexity::

TOC

Description

Complexity is determined by the number of decision points in a method plus one for the
method entry.  The decision points are 'if', 'while', 'for', and 'case labels'.  Generally,
1-4 is low complexity, 5-7 indicates moderate complexity, 8-10 is high complexity,
 and 11+ is very high complexity.
Message
The {0} ''{1}'' has a Cyclomatic Complexity of {2}.
Class
net.sourceforge.pmd.rules.CyclomaticComplexity
Reference
-
ExternalInfoUrl
http://pmd.sourceforge.net/rules/codesize.html#CyclomaticComplexity
TypeResolution
-
Priority
3

Example

Example code snippet of this rule. 

// Cyclomatic Complexity = 12
public class Foo {
1   public void example()  {
2       if (a == b)  {
3           if (a1 == b1) {
                fiddle();
4           } else if a2 == b2) {
                fiddle();
            }  else {
                fiddle();
            }
5       } else if (c == d) {
6           while (c == d) {
                fiddle();
            }
7        } else if (e == f) {
8           for (int n = 0; n < h; n++) {
                fiddle();
            }
        } else{
            switch (z) {
9               case 1:
                    fiddle();
                    break;
10              case 2:
                    fiddle();
                    break;
11              case 3:
                    fiddle();
                    break;
12              default:
                    fiddle();
                    break;
            }
        }
    }
}

Properties

Following properties are defined:
Name
reportLevel
Description
The Cyclomatic Complexity reporting threshold
Value

Rule ExcessivePublicCount::

TOC

Description

A large number of public methods and attributes declared in a class can indicate the
 class may need to be broken up as increased effort will be required to thoroughly test it.
Message

Rule TooManyFields::

TOC

Description

Classes that have too many fields could be redesigned to have fewer fields, possibly
 through some nested object grouping of some of the information.  For example, a class with
 city/state/zip fields could instead have one Address field.
Message
Too many fields
Class
net.sourceforge.pmd.rules.design.TooManyFields
Reference
-
ExternalInfoUrl
http://pmd.sourceforge.net/rules/codesize.html#TooManyFields
TypeResolution
-
Priority
3

Example

Example code snippet of this rule. 
   
public class Person {
   String one;
   int two;
   int three;
   [... many more public fields ...]
}
