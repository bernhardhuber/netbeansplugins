Rule IntegerInstantiation::

TOC

Description

In JDK 1.5, calling new Integer() causes memory allocation.  Integer.valueOf() is more memory friendly.
Message
Avoid instantiating Integer objects. Call Integer.valueOf() instead.
Class
net.sourceforge.pmd.rules.XPathRule
Reference
-
ExternalInfoUrl
-
TypeResolution
-
Priority
2

Example

Example code snippet of this rule. 
  
public class Foo {
 private Integer i = new Integer(0); // change to Integer i = Integer.valueOf(0);
}

xpath

    or ClassOrInterfaceType/@Image='java.lang.Integer')]
