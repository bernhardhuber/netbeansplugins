Rule NoLongScripts::

TOC

Description

 Scripts should be part of Tag Libraries, rather than part of JSP pages.
Message
Avoid having long scripts (e.g. Javascript) inside a JSP file.
Class
net.sourceforge.pmd.rules.XPathRule
Reference
-
ExternalInfoUrl
http://pmd.sourceforge.net/rules/basic-jsp.html#NoLongScripts
TypeResolution
-
Priority
2

Example

Example code snippet of this rule. 
			
<HTML>
<BODY>
<!--Java Script-->
<SCRIPT language="JavaScript" type="text/javascript">
<!--
function calcDays(){
  var date1 = document.getElementById('d1').lastChild.data;
  var date2 = document.getElementById('d2').lastChild.data;
  date1 = date1.split("-");
  date2 = date2.split("-");
  var sDate = new Date(date1[0]+"/"+date1[1]+"/"+date1[2]);
  var eDate = new Date(date2[0]+"/"+date2[1]+"/"+date2[2]);
  var daysApart = Math.abs(Math.round((sDate-eDate)/86400000));
  document.getElementById('diffDays').lastChild.data = daysApart;
}

onload=calcDays;
//-->
</SCRIPT>
</BODY>
</HTML>

Rule NoScriptlets::

TOC

Description

 Scriptlets should be factored into Tag Libraries or JSP declarations,
			rather than being part of JSP pages.
Message
Avoid having scriptlets inside a JSP file.
Class
net.sourceforge.pmd.rules.XPathRule
Reference
-
ExternalInfoUrl
http://pmd.sourceforge.net/rules/basic-jsp.html#NoScriptlets
TypeResolution
-
Priority
3

Example

Example code snippet of this rule. 
			 
<HTML>
<HEAD>
<%
response.setHeader("Pragma", "No-cache");
%>
</HEAD>
	<BODY>
		<jsp:scriptlet>String title = "Hello world!";</jsp:scriptlet>
	</BODY>
</HTML>

Properties

Following properties are defined:
Name
xpath
Description
-
Value
					 //descendant::*[ 
self::JspScriptlet  or (self::Element and string:upper-case(@Name)="JSP:SCRIPTLET") ]

Rule NoInlineStyleInformation::

TOC

Description

 Style information should be put in CSS files, not in JSPs.
			Therefore, don't use <B> or <FONT> tags, or attributes like "align='center'".
Message
Avoid having style information in JSP files.
Class
net.sourceforge.pmd.jsp.rules.NoInlineStyleInformation
Reference
-
ExternalInfoUrl
http://pmd.sourceforge.net/rules/basic-jsp.html#NoInlineStyleInformation
TypeResolution
-
Priority
3

Example

Example code snippet of this rule. 
			 
<html><body><p align='center'><b>text</b></p></body></html>

Rule IframeMissingSrcAttribute::

TOC

Description

		  IFrames which are missing a src element can cause security information popups 
		  in IE if you are accessing the page through SSL. See http://support.microsoft.com/default.aspx?scid=kb;EN-US;Q261188
Message
IFrames must have a src attribute.
Class
net.sourceforge.pmd.rules.XPathRule
Reference
-
ExternalInfoUrl
http://pmd.sourceforge.net/rules/basic-jsp.html#IframeMissingSrcAttribute
TypeResolution
-
Priority
2

Example

Example code snippet of this rule. 
			
<HTML><title>bad example><BODY>
<iframe></iframe>
</BODY> </HTML>

<HTML><title>good example><BODY>
<iframe src="foo"></iframe>
</BODY> </HTML>

Properties

Following properties are defined:
Name
xpath
Description
-
Value
					 
	//Element[string:upper-case(@Name)="IFRAME"][count(Attribute[string:upper-case(@Name)="SRC" ]) = 0]

Rule NoHtmlComments::

TOC

Description

		  In a production system, HTML comments increase the payload between the application 
		  server to the client, and serve little other purpose. Consider switching to 
		  JSP comments.
Message
Use JSP comments instead of HTML comments
Class
net.sourceforge.pmd.rules.XPathRule
Reference
-
ExternalInfoUrl
http://pmd.sourceforge.net/rules/basic-jsp.html#NoHtmlComments
TypeResolution
-
Priority
2

Example

Example code snippet of this rule. 
			
<HTML><title>bad example><BODY>
<!-- HTML comment -->
</BODY> </HTML>

<HTML><title>good example><BODY>
<%-- JSP comment --%>
</BODY> </HTML>

xpath
Description
-
Value
					 
	//CommentTag
