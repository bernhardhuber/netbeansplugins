Rule AvoidCatchingThrowable::

TOC

Description

This is dangerous because it casts too wide a net; it can catch things like OutOfMemoryError.
Message
A catch statement should never catch throwable since it includes errors.

Example

Example code snippet of this rule. 
                
public class Foo {
 public void bar() {
  try {
   // do something
  } catch (Throwable th) {  //Should not catch throwable
   th.printStackTrace();
  }
 }
}

Rule ExceptionAsFlowControl::

TOC

Description

          Using Exceptions as flow control leads to GOTOish code and obscures true exceptions when debugging.
Message
Avoid using exceptions as flow control.
Class
net.sourceforge.pmd.rules.design.ExceptionAsFlowControl
Reference
-
ExternalInfoUrl
http://pmd.sourceforge.net/rules/strictexception.html#ExceptionAsFlowControl
TypeResolution
-
Priority
3

Example

Example code snippet of this rule. 
  
public class Foo {
 void bar() {
  try {
   try {
   } catch (Exception e) {
    throw new WrapperException(e);
    // this is essentially a GOTO to the WrapperException catch block
   }
  } catch (WrapperException e) {
   // do some more stuff
  }
 }
}

Rule AvoidThrowingNullPointerException::

TOC

Description

Avoid throwing a NullPointerException - it's confusing because most people will assume that the
