Rule ShortVariable::

TOC

Description

Detects when a field, local, or parameter has a very short name.
Message
Avoid variables with short names like {0}
Class
net.sourceforge.pmd.rules.XPathRule
Reference
-
ExternalInfoUrl
http://pmd.sourceforge.net/rules/naming.html#ShortVariable
TypeResolution
-
Priority
3

Example

Example code snippet of this rule. 

public class Something {
  private int q = 15; // VIOLATION - Field
  public static void main( String as[] ) {  // VIOLATION - Formal
    int r = 20 + q; // VIOLATION - Local
    for (int i = 0; i < 10; i++) { // Not a Violation (inside FOR)
      r += q;
    }
  }
}

Example code snippet of this rule.

Example code snippet of this rule.

indicates a field and thus is confusing.
      




Message


Avoid naming non-fields with the prefix 'm_'

Reference


-


ExternalInfoUrl


http://pmd.sourceforge.net/rules/naming.html#MisleadingVariableName


TypeResolution


-


Priority


3




Example


Example code snippet of this rule.

    private int m_foo; // OK
    public void bar(String m_baz) {  // Bad
      int m_boz = 42; // Bad
    }
  }

xpath
